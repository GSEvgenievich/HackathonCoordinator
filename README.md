# Hackathon Coordinator 🚀

**Платформа для управления IT-соревнованиями с командным режимом** — десктопное приложение для организаторов, капитанов и участников хакатонов. Обеспечивает полный цикл координации: от создания соревнования до подведения итогов, с интеграцией задач, чатов и GitHub.

## Ключевые возможности

### 🏆 Управление соревнованиями
- Создание, редактирование и мониторинг хакатонов
- **Построение расписания с этапами** (автоматический расчет времени)
- Архивация завершенных соревнований с очисткой данных

### 👥 Управление командами и ролями
- Система приглашений по уникальному коду
- Назначение капитанов и гибкая система должностей (`Positions`)
- Автоматическая фиксация состава команды при подведении итогов

### 📋 Управление задачами
- Создание, назначение и отслеживание прогресса задач
- **Жизненный цикл задачи:** `В планах` → `В процессе` → `На проверке` → `Завершена`
- Двустороннее подтверждение выполнения (участник запрашивает, капитан утверждает)

### 💬 Встроенная коммуникация (Real-time)
- **Командные чаты** и **чаты задач** на базе SignalR
- Отправка файлов и изображений с генерацией превью (хранилище MinIO)
- Умные упоминания `@notify` для срочных уведомлений

### 🔗 Интеграция с GitHub
- OAuth 2.0 аутентификация
- Автоматическое создание репозиториев и веток
- Приглашение участников и организаторов в репозиторий с разными правами (`push`/`pull`)

### 🔔 Умные уведомления (Real-time)
- **Фоновые сервисы:** автоматическое оповещение о старте соревнований, этапов и приближении дедлайнов
- Уведомления о новых задачах, изменениях статусов, сообщениях в чатах

### 📊 Подведение итогов
- Визуальное Drag-and-Drop определение мест
- Добавление комментариев для каждой команды
- **Экспорт результатов в PDF** и детализированный отчет в Excel (команды, задачи, участники, прогресс)

### 🎨 Пользовательский опыт
- **6 цветовых тем**: Светлая, Темная, Весна, Лето, Осень, Зима
- Адаптивный интерфейс с боковой панелью и реактивной навигацией

### 🔐 Безопасность
- JWT-аутентификация с refresh-механизмом
- **AES-256 шифрование** GitHub-токенов в базе данных
- **DPAPI шифрование** JWT-токена на клиенте
- Разграничение прав доступа на уровне ролей (Admin, Organizer, Captain, Member)

## Архитектура

### Трехзвенная архитектура с асинхронными взаимодействиями:

```
┌─────────────────┐     HTTP/JSON     ┌─────────────────┐     EF Core      ┌─────────────────┐
│                 │ ◄────────────────► │                 │ ◄───────────────► │                 │
│    WPF Client   │     REST API      │  ASP.NET Core   │    Database      │     MySQL 8     │
│   (MVVM)        │                   │     Web API     │   Connection     │                 │
│                 │                   │                 │                   │                 │
└────────┬────────┘                   └────────┬────────┘                   └─────────────────┘
         │                                      │
         │      SignalR (WebSocket)            │
         └──────────────────────────────────────┘
                    (Чат, уведомления)
         
┌─────────────────┐     AWS S3 API    ┌─────────────────┐
│    WPF Client   │ ◄────────────────► │      MinIO      │
│                 │   Presigned URL   │  (Файловое      │
│                 │                   │   хранилище)    │
└─────────────────┘                   └─────────────────┘
```

### Ключевые компоненты системы:
- **WPF Client:** MVVM, кастомные конвертеры, стили, SignalR клиент
- **Web API:** RESTful контроллеры, фоновые сервисы, SignalR хабы
- **Service Layer:** DTO для обмена данными между клиентом и сервером
- **MinIO:** S3-совместимое хранилище для файлов и изображений

## Быстрый старт (Docker)

Это самый простой способ запустить серверную часть.

### 1. Клонируйте репозиторий
```bash
git clone https://github.com/GSEvgenievich/HackathonCoordinator
cd HackathonCoordinator
```

### 2. Настройте конфигурацию
```bash
cp .env.example .env
# Отредактируйте .env (хотя бы замените GitHub OAuth данные, если нужно)
```

### 3. Запустите все сервисы одной командой
```bash
docker compose up -d
```

### 4. Проверьте, что всё работает
- **API + Swagger:** [http://localhost:5046/swagger](http://localhost:5046/swagger)
- **phpMyAdmin (для БД):** [http://localhost:8081](http://localhost:8081) (сервер: `db`, логин/пароль из `.env`)
- **MinIO Console (для файлов):** [http://localhost:9001](http://localhost:9001) (логин/пароль: `minioadmin`/`minioadmin`)

### 5. Запустите WPF клиент
- Откройте решение `HackathonCoordinator.sln` в Visual Studio
- Соберите и запустите проект `HackathonCoordinator.WPFClient`

## Локальная разработка

### Предварительные требования
- .NET 8 SDK
- Visual Studio 2022 (с нагрузкой `.NET desktop development`)
- MySQL 8.x (или Docker для БД)

### Шаги для запуска

1. **Клонируйте репозиторий**
   ```bash
   git clone https://github.com/GSEvgenievich/HackathonCoordinator
   ```

2. **Настройте базу данных MySQL** (создайте БД и пользователя)
   ```sql
   CREATE DATABASE HackathonCoordinatorDb;
   CREATE USER 'hackathon_user'@'localhost' IDENTIFIED BY 'UserPassword123!';
   GRANT ALL PRIVILEGES ON HackathonCoordinatorDb.* TO 'hackathon_user'@'localhost';
   ```
   - Или выполните скрипт `init.sql` в вашей СУБД.

3. **Настройте строку подключения** в `HackathonCoordinator.WebAPI/appsettings.Development.json`
   ```json
   "ConnectionStrings": {
     "DefaultConnection": "Server=localhost;Port=3306;Database=HackathonCoordinatorDb;User=hackathon_user;Password=UserPassword123!;"
   }
   ```

4. **Примените миграции и запустите API**
   ```bash
   cd HackathonCoordinator.WebAPI
   dotnet ef database update
   dotnet run
   ```

5. **Запустите WPF клиент**
   - Откройте `HackathonCoordinator.sln` в Visual Studio
   - Установите `HackathonCoordinator.WPFClient` стартовым проектом
   - Нажмите `F5`

## Конфигурация

### Основные параметры в файле `.env`

```env
# ============================================
# НАСТРОЙКИ БАЗЫ ДАННЫХ (MySQL)
# ============================================

# Пароль для root-пользователя MySQL
DB_ROOT_PASSWORD=RootPassword123!

# Имя базы данных, которая будет создана
DB_NAME=HackathonCoordinatorDb

# Имя пользователя для подключения к БД (не root)
DB_USER=hackathon_user

# Пароль пользователя БД
DB_PASSWORD=UserPassword123!


# ============================================
# НАСТРОЙКИ API
# ============================================

# Среда выполнения (Development или Production)
ASPNETCORE_ENVIRONMENT=Production


# ============================================
# НАСТРОЙКИ JWT АУТЕНТИФИКАЦИИ
# ============================================

# Секретный ключ для подписи JWT-токенов (должен быть сложным)
JWT_KEY=hackathon_c00rd_key_super_194_key_inator_13610

# Издатель токена (обычно домен приложения)
JWT_ISSUER=HackathonCoordinator

# Получатель токена (обычно домен приложения)
JWT_AUDIENCE=HackathonUsers

# Время жизни токена в минутах (2 часа)
JWT_EXPIRE_MINUTES=120


# ============================================
# НАСТРОЙКИ GitHub OAuth
# ============================================

# Client ID из зарегистрированного OAuth приложения GitHub
GITHUB_CLIENT_ID=YOUR_GITHUB_CLIENT_ID

# Client Secret из зарегистрированного OAuth приложения GitHub
GITHUB_CLIENT_SECRET=YOUR_GITHUB_CLIENT_SECRET

# URL для перенаправления после авторизации в GitHub
GITHUB_REDIRECT_URI=http://YOUR_DOMEN_NAME/api/oauth/github-callback


# ============================================
# НАСТРОЙКИ ШИФРОВАНИЯ
# ============================================

# Ключ для AES-256 шифрования GitHub-токенов
# ⚠️ ВАЖНО: Не менять после создания базы данных!
ENCRYPTION_KEY=1my1-02ro1-irdnaa-56dj8y5td6-key-32-10fa2


# ============================================
# НАСТРОЙКИ MINIO (Файловое хранилище)
# ============================================

# Логин администратора MinIO
MINIO_ROOT_USER=minioadmin

# Пароль администратора MinIO
MINIO_ROOT_PASSWORD=minioadmin

# Внутренний адрес MinIO в Docker-сети
MINIO_ENDPOINT=minio:9000

# Публичный адрес MinIO для доступа из WPF-клиента
MINIO_PUBLIC_ENDPOINT=localhost:9000

# Название бакета (контейнера) для файлов
MINIO_BUCKET=hackathon-files

# Использовать SSL (для локальной разработки false)
MINIO_USE_SSL=false
```

## Интеграция с GitHub

### Настройка OAuth приложения:

1. Перейдите в [GitHub Developer Settings → OAuth Apps](https://github.com/settings/developers)
2. Нажмите **"New OAuth App"**
3. Заполните поля:
   - **Application name:** `Hackathon Coordinator`
   - **Homepage URL:** `http://localhost:5046` (или ваш домен)
   - **Authorization callback URL:** `http://localhost:5046/api/oauth/github-callback` (важно!)
4. После создания скопируйте `Client ID` и сгенерируйте `Client Secret`
5. Вставьте их в файл `.env` и перезапустите контейнеры:
   ```bash
   docker compose restart api
   ```

### Что умеет интеграция:

| Функция | Кто может использовать | Описание |
|---------|------------------------|----------|
| Привязка аккаунта | Все пользователи | OAuth вход через GitHub |
| Создание репозитория | Капитан команды | Автоматическое создание репозитория в аккаунте капитана |
| Создание веток | Капитан | Автоматическое создание ветки для задачи |
| Приглашение участников | Капитан | Добавление участников команды в репозиторий с правами `push` |
| Приглашение организаторов | Капитан | Добавление организаторов с правами `pull` для контроля |

## Полезные команды

### Docker
```bash
# Запуск всех сервисов в фоне
docker compose up -d

# Остановка всех сервисов
docker compose down

# Остановка с удалением томов (очистка БД и файлов)
docker compose down -v

# Просмотр логов API
docker logs hackathon-api -f

# Пересборка и запуск
docker compose up -d --build
```

### Entity Framework Core (из папки `HackathonCoordinator.WebAPI`)
```bash
# Применение миграций к БД
dotnet ef database update

# Создание новой миграции (после изменения моделей)
dotnet ef migrations add MigrationName

# Удаление последней миграции
dotnet ef migrations remove

# Генерация SQL скрипта миграции
dotnet ef migrations script
```

### .NET CLI
```bash
# Сборка всего решения
dotnet build

# Запуск API в режиме разработки с автоперезагрузкой
dotnet watch run --project HackathonCoordinator.WebAPI

# Запуск всех тестов
dotnet test
```
